## 🛠️ План восстановления (обязательно!)

### Шаг 1: Повтор теории (жёстко)
Нужно **переучить** три вещи:

1. **`is` vs `==`** — это не одно и то же
   - `==` сравнивает **значения** (содержимое)
   - `is` сравнивает **идентичность** (это один и тот же объект в памяти?)

2. **`id()` и адреса памяти** — каждый объект имеет уникальный адрес
   - `id(a)` возвращает адрес объекта в памяти
   - разные списки с одинаковым содержимом имеют РАЗНЫЕ `id()`

3. **Интернирование** — Python кэширует некоторые объекты для оптимизации
   - `a = 5; b = 5` → `a is b` == True (малые числа закэшированы)
   - `a = []; b = []` → `a is b` == False (списки не кэшируются)

### Шаг 2: Практика (код + примеры)
Нужно написать и **запустить** примеры:

```python
# Пример 1: is vs ==
a = [1, 2, 3]
b = [1, 2, 3]

print(f"a == b: {a == b}")      # True (содержимое одинаковое)
print(f"a is b: {a is b}")      # False (разные объекты!)
print(f"id(a): {id(a)}")
print(f"id(b): {id(b)}")
```

```python
# Пример 2: интернирование
x = 5
y = 5
print(f"x is y: {x is y}")      # True! (числа кэшируются)

# Но:
x = 300
y = 300
print(f"x is y: {x is y}")      # Может быть False (кэш работает до ~256)
```
